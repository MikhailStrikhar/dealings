# README_AI.md

## О проекте

Это прикладная конфигурация 1С для учёта инвестиционных операций, позиций, рыночных данных и финансового результата.

Код должен соответствовать возможностям и ограничениям платформы 1С:Предприятие.

## Общие правила для AI

- Писать код только на встроенном языке 1С.
- Использовать русские имена процедур, функций и переменных в стиле проекта.
- Предлагать практичные решения, применимые в реальной разработке 1С.
- Не предлагать абстрактные классы, dependency injection, сложные шаблоны из Java/C#/Python без явной необходимости.
- Если контекста недостаточно, лучше сначала предложить краткий план или 2-3 реалистичных варианта.

## Использование БСП

В конфигурацию включена БСП (Библиотека стандартных подсистем) от фирмы 1С.

Правила:
- по возможности использовать механизмы и функции БСП вместо собственной реализации
- не дублировать функциональность, которая уже есть в БСП
- объекты и код БСП изменять нельзя
- если требуется расширение поведения, использовать собственные модули или расширение конфигурации

## Ограничения платформы

Решения должны быть совместимы с платформой 1С.

Не использовать:
- конструкции и подходы, отсутствующие во встроенном языке 1С
- абстрактные классы и DI-контейнеры
- архитектурные паттерны, не дающие практической пользы в 1С
- решения, завязанные на возможности других языков и рантаймов

## Принцип минимальных изменений

При изменении кода:
- не переписывать модуль целиком без необходимости
- изменять только те участки, которые действительно относятся к задаче
- не менять публичные интерфейсы без необходимости
- сохранять существующую бизнес-логику, если задача не требует её изменения

## Как работать с задачами

Перед существенными изменениями:
- сначала предложить краткий план
- показать, какие модули будут затронуты
- отдельно отметить возможные риски

При рефакторинге:
- не смешивать архитектурную перестройку и прикладную доработку в одной задаче
- по возможности делать точечные, безопасные изменения
- после изменений кратко перечислять изменённые файлы и суть изменений

## Что важно учитывать

- Это проект 1С, а не проект на Python, Java или C#.
- Нужно учитывать ограничения платформы и практику разработки 1С.
- Решения должны быть понятными, поддерживаемыми и реалистичными для прикладной конфигурации.
- Подробная карта подсистем и ответственности описана в `PROJECT_MAP.md`.
- Стандарты разработки описаны в `DEV_STANDARDS.md`.
