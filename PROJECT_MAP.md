# PROJECT_MAP.md

## Назначение

Конфигурация предназначена для учёта инвестиционных операций, позиций по инструментам, текущих цен, свечей и финансового результата.

## Подсистемы

### 1. Учёт операций

Основные объекты:
- Документ БиржеваяОперация
- Документ ВводОстатков
- Регистр накопления ПозицииИнструментов
- Регистр накопления ФинансовыеРезультатыПоСделкам

Назначение:
- отражение покупки и продажи инструментов
- хранение текущих позиций
- расчёт финансового результата
- обработка комиссий и связанных параметров операции

### 2. Интеграция с брокерским API

Основные модули:
- ИнтеграцияTInvestAPI
- ИнтеграцияTInvestОперации
- ИнтеграцияTInvestРыночныеДанные

Назначение:
- выполнение HTTP-запросов к API
- получение операций брокера
- загрузка инструментов
- загрузка текущих цен и других рыночных данных

### 3. Рыночные данные

Основные объекты:
- Регистр сведений ТекущиеЦены
- Регистр сведений СвечиИнструментов
- Регистр накопления ИсторияЦенИнструментов

Основные модули:
- СвечиСервис

Назначение:
- загрузка последних цен
- загрузка исторических свечей
- хранение данных для анализа динамики цен

### 4. Бизнес-логика сделок

Основные модули:
- СделкиСервис

Назначение:
- проверки прикладных условий
- расчёт позиции
- расчёт финансового результата
- подготовка логики, используемой при проведении документов

## Распределение ответственности

### ИнтеграцияTInvestAPI
Отвечает только за:
- HTTP-соединение
- URL
- заголовки
- вызовы методов внешнего API
- низкоуровневую обработку ответа

### ИнтеграцияTInvestОперации
Отвечает за:
- получение операций брокера
- разбор ответа API
- преобразование данных API в структуры проекта

### ИнтеграцияTInvestРыночныеДанные
Отвечает за:
- загрузку текущих цен
- загрузку справочных данных по инструментам
- координацию сценариев, связанных с рыночными данными

### СвечиСервис
Отвечает за:
- отбор инструментов для загрузки
- вызовы API загрузки свечей
- преобразование и запись свечей в регистр

### СделкиСервис
Отвечает за:
- прикладные проверки
- расчёт позиций
- расчёт финансового результата
- вспомогательную бизнес-логику по операциям

### Документ БиржеваяОперация
Отвечает за:
- проведение документа
- формирование движений по регистрам

## Основные потоки данных

### Загрузка операций
T-Invest API
-> ИнтеграцияTInvestAPI

-> объекты и данные конфигурации

### Загрузка последних цен
T-Invest API
-> ИнтеграцияTInvestAPI
-> ИнтеграцияTInvestРыночныеДанные
-> Регистр сведений ТекущиеЦены

### Загрузка свечей
T-Invest API
-> ИнтеграцияTInvestAPI
-> СвечиСервис
-> Регистр сведений СвечиИнструментов

### Проведение операции
Документ БиржеваяОперация
-> СделкиСервис
-> Регистр накопления ПозицииИнструментов
-> Регистр накопления ФинансовыеРезультатыПоСделкам

## Ограничения

- Объекты и код БСП изменять нельзя.
- Низкоуровневые HTTP-вызовы не должны смешиваться с бизнес-логикой.
- Модули интеграции не должны напрямую формировать движения регистров.
- Формы и метаданные не изменять без явной задачи.
- Новую функциональность по возможности размещать в существующих подсистемах, а не создавать дублирующие сущности.

## Открытые вопросы архитектуры

- окончательная модель обработки шорт-сделок
- место расчёта финансового результата в сложных сценариях
- возможная поддержка нескольких брокерских платформ
